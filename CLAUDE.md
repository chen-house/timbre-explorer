# CLAUDE.md

这个文件是给 Claude Code 在后续开发会话中的指引。

## 项目背景

这是一个面向初学者（非零基础）的中文声音设计学习站。读者是想成为声音设计师/混音师、有一定实操好奇心但经验少的人。

## 核心原则

### 内容写作

1. **先核实，再下结论**
   - 不猜测、不编造
   - 每个数字、每条法则都要有依据：物理规律 / 心理声学 / 行业标准 / 个人实战经验（明确标注）
   - 例：电话声引用 `ITU-T G.712 (300–3400 Hz)`，不只是"中频"
   - 例：人耳敏感频段引用 `ISO 226 等响曲线 + 外耳道共振 ~3 kHz`

2. **专业 + 通俗并存**
   - 不要为了显专业堆术语
   - 也不要为了通俗丢失精度
   - 例如不说"EQ 让声音更好听"，说"EQ 调整频率能量分布，从而改变音色"

3. **理论 + 实操双管齐下**
   - 每个理论点必须配实操动作（具体频率、具体 dB、具体场景）
   - 例：不说"高频会让声音更亮"，说"在 10 kHz 加 +3 dB High Shelf 给人声添加空气感"

4. **修正性 vs 创意性 是关键思维框架**
   - 凡涉及处理决策，先问：是要解决问题，还是塑造性格？
   - 这是职业混音师与新手的核心认知差异

### 视觉/交互设计

- **极简 + 手绘风** — 不加阴影、渐变、纸张纹理等拟物特效
- **手绘感来自**：简单几何 + 圆角端点 + 不规则但克制的曲线
- **不用 emoji** 作为装饰元素（内容本身需要的除外）
- **配色**：暖橙 `#D85A30` 主，青绿 `#1D9E75` 次，紫色 `#7B5EA7` 辅
- **字体**：Noto Sans SC（思源黑体），最小 11 px
- **音频演示尽可能实时合成** — 用 Web Audio API，让参数真实驱动声音

### 代码规范

- 纯静态 HTML/CSS/JS，无构建步骤
- 每个页面独立可运行
- 共享样式在 `assets/css/main.css`
- 页面专属样式独立成 `assets/css/<page>.css`
- 互动逻辑独立成 `assets/js/<page>.js`
- HTML 语义化：`<section>` `<header>` `<nav>` `<footer>`
- 注释保持精炼，标记区块用即可

## 开发流程

### 添加新效果器文章

1. 在 `effects/` 下复制 `eq.html` 作为模板
2. 替换 `<title>`、`<header>` 内容
3. 替换 `<link>` 引入的 CSS 文件
4. 替换 `<script>` 引入的 JS 文件
5. 编写对应的 `assets/css/<name>.css`
6. 编写对应的 `assets/js/<name>.js`（如有互动）
7. 更新 `effects/index.html` 中该效果器的状态标签
8. 更新 `index.html` 中"已收录 N 篇"

### 添加新板块

1. 创建 `<topic>/` 目录和 `index.html`
2. 复制现有板块 `index.html` 作为模板
3. 在所有页面的 `<nav class="site-nav">` 内添加导航链接
4. 在 `index.html` 首页"Three Pillars"区域添加第 N 张卡片

## 建议优先级

按照学习路径自然顺序：

1. ✅ EQ 均衡器（已完成）
2. ⬜ Filter 滤波器（EQ 的延伸，最容易接续）
3. ⬜ Reverb 混响（空间感，独立模块）
4. ⬜ Delay 延迟（与 Reverb 对照，时间类效果器）
5. ⬜ Saturator 饱和器（最后讲，因为依赖谐波概念）
6. ⬜ 音色板块（依赖 EQ 与 Saturator 的概念）
7. ⬜ 氛围色板块（依赖前面所有内容）

## 不要做什么

- ❌ 不要在响应式布局里加复杂的媒体查询，移动端简单堆叠即可
- ❌ 不要引入框架（React/Vue 等），保持纯静态
- ❌ 不要引入图标字体库（Font Awesome 等），手绘 SVG 即可
- ❌ 不要在文章里堆砌长串 emoji
- ❌ 不要添加"返回顶部""分享"等花哨按钮，先把内容做扎实
- ❌ 不要为了"看起来丰富"加无关的装饰元素

## 互动音频演示的设计原则

- 使用 Web Audio API 实时生成，不依赖音频文件
- 粉噪声（pink noise）适合训练耳朵 — 因为它的能量分布贴近人耳听觉
- 音乐性内容用简单和弦合成 — 让 EQ/效果器的处理在音乐上下文里也能听清
- 提供 A/B 对比开关 — 这是混音师训练耳朵的核心方法
- 提供"预设" — 让用户能快速跳到典型场景（电话声、空气感、扫频）
- 视觉化曲线必须实时响应参数变化
